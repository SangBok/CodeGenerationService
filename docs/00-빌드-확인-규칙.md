# 빌드 확인 규칙(Agent)

## 목적
- 레거시 `csharp/AutomationTemplate`는 Visual Studio(.NET Framework 4.8) 기반이어서, 코드 수정 후 컴파일/빌드 오류를 조기에 발견하기 위함입니다.

## 기본 원칙
1. 소스 코드(기능/구조)가 변경되면, 변경 범위에 해당하는 빌드를 항상 실행합니다.
2. 빌드 실행 결과는 “오류 0 / 성공” 상태인지 확인합니다.
3. 빌드가 실패하면, 실패 원인을 수정한 뒤 **동일 빌드를 재실행**해서 다시 성공을 확인합니다.

## 적용 범위

### A) 레거시 WinForms: `AutomationTemplate`
- 빌드 명령(Visual Studio MSBuild 사용):
  - `C:\Program Files\Microsoft Visual Studio\2022\Community\MSBuild\Current\Bin\MSBuild.exe "d:\1_Git\CodeGenerationService\csharp\AutomationTemplate\AutomationTemplate.sln" /t:Build /p:Configuration=Debug /p:Platform="Any CPU"`
- 확인 기준:
  - 빌드 로그에 `error`가 없고 최종 요약에서 `성공 >= 0, 실패 = 0`이어야 합니다.

### B) 웹 편집기: `web/bt-editor`
- `npm run build`를 실행해서 TypeScript/Vite 빌드가 정상인지 확인합니다.

### C) PoC 런타임(.NET 6): `bt/runtime`
- `dotnet build BtRuntime.sln -c Release`로 빌드가 정상인지 확인합니다.

## 기록
- 이 문서는 “규칙 문서”로 남기며, 이후 코드 수정 후 빌드 검증을 반복 실행합니다.
