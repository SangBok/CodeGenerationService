# API 명세서

## 1. 목적

설비 제어 코드 자동 생성 서비스의 **코드 생성 API**에 대한 요청/응답·에러·인증 등을 문서화한다. 구현 시 참고용이며, 구체적 엔드포인트·프로토콜은 구현 단계에서 확정한다.

---

## 2. 코드 생성 API (개념)

### 2.1 역할

- 클라이언트가 **하드웨어 명세**와 **동작 명세(UML 추출 텍스트 또는 Draw.io 관련 데이터)** 를 전달하면, 서비스가 **레이어드 C# 제어 코드**를 생성하여 반환한다.
- 생성은 **레이어별 단계**(Step 1 ~ 5)로 수행될 수 있으며, API는 "전체 한 번에" 또는 "단계별 호출" 중 하나 또는 둘 다를 지원하도록 설계할 수 있다.

### 2.2 요청 (예시)

- **메서드**: POST (가정).
- **본문 예시 구조**:
  - `hardwareSpec`: 하드웨어 명세 (문서 [03-하드웨어명세-스키마](03-하드웨어명세-스키마.md)에 맞는 JSON 등).
  - `behaviorSpec`: 동작 명세 (UML 추출 텍스트 또는 Draw.io XML/구조화 데이터).
  - `options` (선택): 생성 옵션 (예: 레이어만 지정, .NET 버전, 출력 형식 등).

### 2.3 응답 (예시)

- **성공 시**:
  - `files`: 생성된 파일 목록. 각 항목: `path`(상대 경로), `content`(파일 내용).
  - 또는 `projectStructure`: 프로젝트/솔루션 구조와 파일 목록.
- **실패 시**: 에러 코드·메시지 (4절 참고).

### 2.4 인증

- 온프레미스 환경 및 보안 요구에 따라 **내부 인증**(API Key, AD/LDAP 연동 등) 필요 여부를 구현 단계에서 확정한다.
- 인증 방식이 정해지면 이 문서에 요청 헤더·토큰 형식을 추가한다.

---

## 3. 단계별 생성 API (선택 사양)

- **역할**: 한 레이어(Step N)만 생성하거나, 이전 단계 출력을 넘겨 다음 단계만 생성하는 API.
- **요청**: `step`(1~5), `hardwareSpec`, `behaviorSpec`, `previousStepOutput`(이전 단계 생성 결과, 선택).
- **응답**: 해당 단계에서 생성된 파일 목록 및 내용.
- 구현 시 레이어별 프롬프트 명세([05-레이어별-코드생성-프롬프트명세](05-레이어별-코드생성-프롬프트명세.md))를 사용한다.

---

## 4. 에러

- **4xx**: 잘못된 요청 (필수 필드 누락, 스키마 불일치, 지원하지 않는 옵션 등).
- **5xx**: 서버/LLM 오류 (Qwen 호출 실패, 타임아웃, 내부 예외 등).
- 응답 본문에 `code`, `message`, (선택) `details` 를 포함하도록 설계할 수 있다.
