# BT 노드 추가 가이드 (PoC)

이 문서는 **웹 BT 편집기에서 노드를 추가**할 수 있게 만드는 것과, **레거시 WinForms(.NET Framework 4.8) 런타임에서 실제로 동작**하게 만드는 것을 함께 설명합니다.

## 용어

- **Node Type**: BT JSON의 `nodes[].type` 값 (예: `Unit.AxisMoveAbsolute`)
- **Node Name**: BT JSON의 `nodes[].name` 값 (없으면 `id`를 사용)
- **Editor Catalog**: 웹 편집기 왼쪽 팔레트에 표시되는 노드 정의 목록
- **Runtime Node**: WinForms 런타임에서 `type`에 따라 실제로 실행되는 C# 구현체

## 전체 흐름(2단계)

1. 웹 편집기 팔레트에 새 노드를 보이게 만들기 (Node Catalog 수정)
2. 레거시 런타임에서 새 노드를 실행되게 만들기 (C# Node 구현 + `BehaviorTree.Build` switch 추가)

## 1) Editor Catalog에 노드 추가하기

다음 파일을 동일하게 수정합니다.

- `[web/bt-editor/public/schemas/node-catalog.poc.json](web/bt-editor/public/schemas/node-catalog.poc.json)`
- `[bt/schema/node-catalog.poc.json](bt/schema/node-catalog.poc.json)`

형식은 `nodes[]` 배열에 아래처럼 `type`과 `parameters`를 추가하면 됩니다.

예) (설명용) 노드 카탈로그 항목 스켈레톤

```json
{
  "type": "Unit.CylinderForward",
  "category": "UnitAction",
  "description": "실린더 전진",
  "parameters": {
    "unit": { "valueType": "string", "required": true },
    "cylinder": { "valueType": "string", "required": true }
  }
}
```

노트:
- 웹 편집기는 기본적으로 `Parameters (JSON)`을 “그대로” 내보냅니다. 따라서 **파라미터 이름/타입은 런타임이 기대하는 키와 맞춰야** 합니다.

## 2) Runtime에서 노드 동작 구현하기

런타임은 다음 파일들을 기준으로 확장합니다.

- 런타임 노드 구현 위치:  
  - `[csharp/AutomationTemplate/AutomationTemplate/5_Utility/BtRuntime/Nodes/](csharp/AutomationTemplate/AutomationTemplate/5_Utility/BtRuntime/Nodes/)`
- BT 인스턴스 생성/해석 위치:  
  - `[csharp/AutomationTemplate/AutomationTemplate/5_Utility/BtRuntime/BehaviorTree.cs](csharp/AutomationTemplate/AutomationTemplate/5_Utility/BtRuntime/BehaviorTree.cs)`

### 2-1) 새 Node 클래스 만들기

아래 3가지만 지키면 됩니다.

1. `INode`를 구현
2. `TickAsync`에서 `context.Emit(..., "tickStart")`와 `context.Emit(..., "tickResult")`를 호출
3. `type`에 맞는 legacy 유닛/동작 메서드를 호출

또한 `parameters`에는 `$ref` 참조가 올 수 있으므로, 기존 런타임에 있는 헬퍼를 사용합니다.

- `BtJson.ResolveInt(token, context.Blackboard)`
- `BtJson.ResolveString(token, context.Blackboard)`

### 2-2) BehaviorTree.Build switch에 추가하기

`BehaviorTree.Build()`의 `switch (def.Type)`에 새 `case`를 추가하고, 새 Node 클래스를 생성하면 됩니다.

예) `Unit.CylinderForward`용 switch 추가 형태(개념 예시)

```csharp
case "Unit.CylinderForward":
    node = new HandlerCylinderForwardNode(
        def.Id,
        def.Name ?? def.Id,
        GetRequiredString(def, "unit"),
        GetRequiredToken(def, "cylinder"));
    break;
```

## 예시 코드: `Unit.CylinderForward`를 Handler 유닛으로 매핑

레거시의 `MHandler`는 `MaterialGrip()`/`MaterialUngrip()`를 제공합니다. 그래서 `cylinder=Gripper`일 때 forward/backward를 각각 매핑할 수 있습니다.

예시(요지):

```csharp
internal sealed class HandlerCylinderForwardNode : INode
{
    public string Id { get; }
    public string Name { get; }
    public string Type => "Unit.CylinderForward";

    public HandlerCylinderForwardNode(string id, string name, string unitName, JToken cylinderToken)
    {
        Id = id;
        Name = name;
        this.unitName = unitName;
        this.cylinderToken = cylinderToken;
    }

    public Task<BtNodeStatus> TickAsync(BtContext context)
    {
        context.Emit(Id, Name, Type, "tickStart");

        var unitObj = context.UnitRegistry.ResolveUnit(unitName);
        var handler = unitObj as MHandler;
        if (handler == null) throw new InvalidOperationException("...");

        var cylinderName = BtJson.ResolveString(cylinderToken, context.Blackboard);
        if (!string.Equals(cylinderName, "Gripper", StringComparison.OrdinalIgnoreCase))
            throw new NotSupportedException("Legacy handler cylinder forward supports only cylinder=Gripper.");

        handler.MaterialGrip();
        context.Emit(Id, Name, Type, "tickResult", BtNodeStatus.Success);
        return Task.FromResult(BtNodeStatus.Success);
    }
}
```

## 사용자가 실제로 노드를 추가하는 방법(끝 사용자 관점)

1. 웹 편집기에서 팔레트에 새 노드가 보이는지 확인
2. 캔버스에 노드를 추가하고, `unit`/`axis`/`positionIdx` 등 `Parameters (JSON)`을 채움
3. Export 후 WinForms에서 BT JSON을 Load하고 Start
