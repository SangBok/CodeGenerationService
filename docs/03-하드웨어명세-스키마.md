# 하드웨어 명세 스키마

## 1. 목적

설비의 하드웨어 정보(축, IO 보드, 로봇, 네트워크/주소, 안전 IO 등)를 **구조화된 형식**으로 정의하여, 코드 생성 서비스의 입력으로 사용하고 System Layer·Hardware Layer 생성 시 참조할 수 있도록 한다.

---

## 2. 포함 항목

### 2.1 기본적으로 다루는 항목

| 항목 | 설명 |
|------|------|
| **축 정보** | 축 개수, 타입, 제조사/프로토콜, 축 ID 등 |
| **IO 보드** | 종류, 개수, 채널 수, 채널 번호 매핑 |
| **로봇** | 제조사, 프로토콜, 대수, 노드/주소 |
| **네트워크/주소** | IP, 포트, 노드 번호 등 (코드 생성 시 상수/설정으로 매핑) |
| **안전/인터록** | 안전 IO, 비상정지, 게이트 등 포함 여부 및 표현 방식 |

### 2.2 추가로 정의할 사항

- **데이터 포맷**: JSON / YAML / Excel 등 표준 스키마 (필드명, 단위, 열거값).
- **제조사/드라이버 매핑**: 사용할 C# 라이브러리·SDK·DLL과의 대응 테이블.
- **유닛–하드웨어 매핑**: 어떤 축/IO/로봇이 어떤 유닛(Mechanical Layer)에 속하는지 (Mechanical Layer 생성 시 참조).

---

## 3. 레이어 매핑

- 이 스키마는 **System Layer** 설정값의 입력이 된다.
- **Hardware Layer**에서 사용할 드라이버/채널 바인딩의 입력이 된다.
- **Mechanical Layer** 생성 시에는 유닛–하드웨어 매핑 부분을 참조한다.

---

## 4. 데이터 포맷 (예시)

아래는 JSON 기반 스키마 예시이다. 실제 필드명·중첩 구조는 프로젝트 규칙에 맞게 조정한다.

### 4.1 설비·축

```json
{
  "equipmentId": "string",
  "axes": [
    {
      "id": "string",
      "name": "string",
      "type": "string",
      "vendor": "string",
      "protocol": "string",
      "nodeOrAddress": "number"
    }
  ]
}
```

### 4.2 IO 보드

```json
{
  "ioBoards": [
    {
      "id": "string",
      "type": "string",
      "vendor": "string",
      "channelCount": "number",
      "channels": [
        { "index": "number", "name": "string", "direction": "input|output" }
      ]
    }
  ]
}
```

### 4.3 로봇

```json
{
  "robots": [
    {
      "id": "string",
      "vendor": "string",
      "protocol": "string",
      "address": "string",
      "port": "number"
    }
  ]
}
```

### 4.4 제조사/드라이버 매핑

- 하드웨어 명세 또는 별도 테이블에서 **제조사·보드 타입** → **사용할 C# 드라이버/라이브러리/DLL** 매핑을 정의.
- 코드 생성 시 Hardware Layer에서 이 매핑에 따라 구현체를 선택한다.

---

## 5. 정리

- 하드웨어 명세 스키마를 이 문서(또는 부록)에서 확정하면, 파싱·검증·프롬프트 설계에서 동일한 구조를 사용할 수 있다.
- 레이어별 코드 생성 시 **03-하드웨어명세-스키마**와 **05-레이어별-코드생성-프롬프트명세**를 함께 참조한다.
